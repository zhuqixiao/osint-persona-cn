# OSINT 浏览器扩展（MV3 v0.3.0）

**安装后日常上网即可**，数据仅写入本机 `http://127.0.0.1:8787`。

## 工作方式

| 模式 | 说明 |
|------|------|
| **被动采集**（默认开） | 访问支持站点时自动记录 URL、标题、停留时长 |
| **队列告警** | 工具栏角标显示待上传条数；上传失败变红 |
| **Web 连接检测** | 弹窗顶部显示本机 Web 是否在线 |
| **API 拦截** | B 站/知乎/GitHub 页面内部分页 API 自动截获（点赞、收藏、发评等） |
| **定时后台同步**（默认开） | 每 4 小时调用本机 `POST /api/ingest/accounts-sync` |
| **右键收录** | 写入行为事件 + 调用 `/api/save` 进知识库 |
| **高停留自动收录** | 内容页停留 ≥90 秒自动进知识库（服务端 `extension.dwell_save_ms` 可调） |

## 支持平台

B站、知乎、GitHub、V2EX、掘金、少数派、虎嗅、36氪、小红书、微博、豆瓣、Twitter/X

（微信仅被动记录已打开的 `mp.weixin.qq.com` 链接，无搜狗搜索行为导入。）

## 安装

1. `osint web` 启动 Web
2. Edge/Chrome → 扩展 → 开发者模式 → **加载已解压** → 选仓库内 `extension/` 目录
3. 弹窗：同步 Cookie → 日常上网 → 完整同步后点「上传浏览采集队列」

## 推荐流程（与 CLI 一致）

1. 扩展弹窗 → **从浏览器同步 Cookie**
2. Web「行为同步」页 → **开始完整同步**（或 CLI `osint sync`）
3. 扩展弹窗 → **上传浏览采集队列**

## 手动操作（可选）

- **上传浏览采集队列**：提交内存中待发送的拦截事件
- **服务端拉取 B站/知乎**：`POST /api/ingest/accounts-sync`
- **浏览器拉取 AICU 发评**：需 `sync.aicu_enabled: true`

## 与 API 导入的关系

- **完整同步**：API bulk + Playwright 补洞（推荐）
- **扩展**：日常增量 + 拦截赞同/发评等 API 补不全的部分

## 隐私

- 只采集本浏览器、本账号访问的页面
- 不上传云端；弹窗可随时关闭采集
